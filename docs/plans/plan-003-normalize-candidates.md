# 実装計画

## 1. 目的とスコープ

本計画は、`scripts/generate-sprites.mjs` の実行中に、画像生成処理の進捗を標準出力へ逐次表示することを目的とします。  
今回のスコープは、開始、対象 `spriteId`、試行回数、再試行、保存完了、失敗、完了集計が標準出力から追えるようにし、関連仕様書とログへ反映するところまでとします。

## 2. 採用アプローチと理由

### 2.1 採用アプローチ

- 生成スクリプトに軽量なログ関数を追加する
- 進行に応じて `stdout` へ人間可読なメッセージを出す
- 致命エラーと再試行可能エラーで出力文言を分ける

### 2.2 理由

1. 長時間の画像生成で現在何が進んでいるか把握しやすくなるため
2. `401/403/429/5xx` や再試行の挙動を目視確認しやすくするため
3. 追加依存なしで安全に導入できるため

## 3. 却下した代替案

### 3.1 GUI 進捗表示を追加する案

- 却下理由: 今回必要なのは CLI 実行時の標準出力であり、GUI は過剰なため

### 3.2 詳細ログを JSON ファイルのみに出す案

- 却下理由: 実行中の見通しが改善されず、今回の要求に対して不足するため

## 4. 制約と前提

- 既存の `generation-errors.json` の役割は維持する
- 成功時の処理フローや `status` 更新条件は変えない
- ログは運用者が読める日本語中心とする
