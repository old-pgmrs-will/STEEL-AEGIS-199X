# 実装計画

## 1. 目的とスコープ

本計画は、残っている必須スプライト資産をすべて生成し、候補選別、正規化、スプライトシート化、メタデータ出力まで完了させることを目的とします。  
今回のスコープは、`sprite-list.csv` 上の全 `required = yes` を `packed` に到達させるところまでとします。

## 2. 採用アプローチと理由

### 2.1 採用アプローチ

- 未生成 `spriteId` は `GPT-image-1.5` で自動生成する
- 生成済み候補は目視で最も安定した1枚を選定する
- 選定画像は自動正規化して `approved/normalized` を作る
- 最後にシート化スクリプトで PNG シートと `sheet-layout.json` を出力する

### 2.2 理由

1. 既存パイプラインを最後まで実運用できるため
2. 完成基準が `packed` で明確なため
3. 実装投入可能な最終成果物まで一気通しで作れるため

## 3. 却下した代替案

### 3.1 途中成果だけで止める案

- 却下理由: ユーザー指示が「残りの作業を全て実行」であり、完了条件を満たさないため

### 3.2 候補選別を行わず機械的に先頭ファイルを採用する案

- 却下理由: 品質低下リスクが高く、現時点では目視選別の方が妥当なため

## 4. 制約と前提

- OpenAI API の利用可能状態に依存する
- 生成画像の品質は一定ではないため、候補選別を挟む
- 背景除去は自動化済みだが、発光表現を一部削る可能性がある

## 5. 実装対象

- 未生成スプライトの生成
- 候補選別
- 正規化
- スプライトシート作成
- メタデータ作成
- ドキュメント反映

## 6. 今回実装しないもの

- 追加アニメーション差分
- 再学習やモデル差し替え
- 手作業レタッチ

## 7. 当初計画からの変更

- `item` と `bullet` を単一シートへまとめる前提は、セルサイズ混在で `pack` が停止するため撤回する
- 最終シート化はカテゴリとセルサイズ単位で自動分割する方針へ変更する
