# 実装計画

## 1. 目的とスコープ

本計画は、`GPT-image-1.5` を利用してスプライト画像の作成工程を自動化し、個別生成からスプライトシート化までを再現可能なパイプラインとして整備することを目的とします。  
今回のスコープは、生成マニフェスト、画像生成呼び出しスクリプト、検収補助、シート化スクリプト、メタデータ出力、運用手順書を揃え、画像ファイルが納品されれば実装へ投入できる状態を作るところまでとします。

## 2. 採用アプローチと理由

### 2.1 採用アプローチ

- OpenAI API を直接呼ぶ Node.js スクリプトを作成する
- 生成対象は `sprite-list.csv` と prompt manifest で管理する
- 画像生成は `raw` へ保存し、検収済み `approved` だけをシート化する
- シート化時に `sheet-layout.json` を自動生成する

### 2.2 理由

1. `spriteId` を単一の台帳で管理でき、生成から実装投入まで追跡しやすいため
2. 個別スプライト生成と後段パックを分離することで、再生成や差し替えが容易なため
3. 画像生成とシート化の責務が明確になり、作業者交代時の引き継ぎが容易なため

## 3. 却下した代替案

### 3.1 スプライトシート全体を一発生成する案

- 却下理由: セル境界崩れ、視点ブレ、サイズ不一致が起きやすく、修正コストが高いため

### 3.2 生成のみ自動化し、台帳やメタデータは手作業で管理する案

- 却下理由: 最終的に不足資産や矩形ずれを起こしやすく、自動化の価値が半減するため

### 3.3 画像処理を外部 GUI ツール前提にする案

- 却下理由: 再現性が弱く、同じ工程を繰り返し実行しにくいため

## 4. 制約と前提

- 画像生成APIの利用には `OPENAI_API_KEY` が必要
- 画像生成結果の品質はモデルとプロンプトに依存するため、最終採用には検収工程を残す
- 背景除去を完全自動にはせず、透過または完全単色背景を前提にする
- 初期対象はプレイヤー、敵、ボス、アイテム、弾の基本アセットに限定する

## 5. 実装対象

- `assets/sprites/` 配下の標準構成
- `sprite-list.csv` と review checklist の初期雛形
- prompt manifest
- `GPT-image-1.5` 呼び出しスクリプト
- シートパッキングと `sheet-layout.json` 生成スクリプト
- 自動化運用ドキュメント

## 6. 今回実装しないもの

- モデルごとの差分最適化
- 画像の高精度背景除去
- アニメーション差分の大量自動生成
- 実時間プレビューGUI
