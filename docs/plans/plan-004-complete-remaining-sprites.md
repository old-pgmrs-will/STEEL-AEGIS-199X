# 実装計画

## 1. 目的とスコープ

本計画は、`incoming/candidates/` に集約された候補スプライト画像を、背景除去とセル正規化によって `approved/normalized/` へ進められるようにすることを目的とします。  
今回のスコープは、自動正規化スクリプトを実装し、現在選定済みの候補画像へ適用し、台帳の状態を更新するところまでとします。

## 2. 採用アプローチと理由

### 2.1 採用アプローチ

- `pngjs` を使った Node.js スクリプトで背景除去とセル正規化を行う
- 背景除去は画像外周からの連結領域を対象とし、背景色近似のピクセルを透過化する
- 正規化後はセルサイズへ収まるよう最近傍補間で縮小し、中央配置で保存する

### 2.2 理由

1. 再現可能で同じ入力から同じ出力を得られるため
2. GUI ツールに依存せずバッチ処理できるため
3. シート化スクリプトへそのまま接続しやすいため

## 3. 却下した代替案

### 3.1 手作業で背景除去する案

- 却下理由: 現時点で対象が増えていく前提のため、再現性と速度が不足するため

### 3.2 しきい値だけで全背景を単純除去する案

- 却下理由: 発光エッジや暗背景に引っ張られて本体まで削るリスクが高いため

## 4. 制約と前提

- 候補画像の背景は完全透過ではなく、灰色や黒系の背景が残っている
- キャラクター周囲に発光があるため、外周連結領域を基準に背景除去する
- 本体の微発光を完全には保持できない場合がある

## 5. 実装対象

- 正規化スクリプト
- `package.json` の npm script
- 台帳更新
- 仕様書更新

## 6. 今回実装しないもの

- 複数フレームアニメーションの一括整列
- 人手による最終レタッチ代替
