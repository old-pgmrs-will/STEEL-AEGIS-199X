# 実装計画

## 1. 目的とスコープ

本計画は、ゲーム実装をコード内ピクセル定義依存から、生成済みの PNG スプライトシートと `sheet-layout.json` を参照する実装へ切り替えることを目的とします。  
今回のスコープは、起動時アセット読込、描画参照、`spriteId` 対応、既存ゲームループへの統合、ビルド確認までとします。

## 2. 採用アプローチと理由

### 2.1 採用アプローチ

- `assets/sprites/manifests/sheet-layout.json` をランタイムで読み込む
- `assets/sprites/sheets/*.png` を Vite のアセット読込で解決する
- 描画クラスは外部シート画像を保持する方式へ切り替える
- `main.ts` でスプライト読込完了後に `Game` を起動する

### 2.2 理由

1. 画像生成済みアセットをそのままゲームに反映できるため
2. 実装とアセット管理の責務を分離できるため
3. 今後の差し替えを `sheet-layout.json` と PNG 更新だけで進めやすくなるため

## 3. 却下した代替案

### 3.1 既存の `PIXEL_SPRITES` を残し、外部シートを補助的に使う案

- 却下理由: 実描画経路が二重化し、最終的な参照元が曖昧になるため

### 3.2 シート画像を `fetch` ベースで個別に読み込む案

- 却下理由: Vite のアセット解決を使った方が構成が単純で、ビルド後のパス解決も安定するため

## 4. 制約と前提

- `sheet-layout.json` にはアンカー情報がないため、ランタイム側で既知スプライトのアンカーを補完する
- 既存のゲーム内 `spriteId` は台帳上の `sprite_id` と一部不一致なので、今回あわせて統一する
- 画像読込は非同期になるため、起動処理を同期コンストラクタのままにはできない

## 5. 実装対象

- 外部スプライトアセットローダー
- `SpriteSheet` の外部画像参照化
- `Game` 初期化シグネチャ変更
- ゲーム内 `spriteId` の統一
- 不要になったコード内ピクセル原稿の撤去

## 6. 今回実装しないもの

- アニメーションフレーム差分
- 自機選択 UI と機体ごとの描画切り替え拡張
- スプライトシートのホットリロード補助機能
